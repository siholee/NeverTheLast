# 적 소환 시스템 개편 문서

## 작성일: 2025-10-27

## 개요
기존의 대기열(Queue) 기반 적 소환 시스템을 완전히 새로운 스테이지/라운드 기반 시스템으로 변경했습니다.

---

## 1. 시스템 구조

### 1.1 스테이지와 라운드
- **스테이지**: 8개의 라운드로 구성
- **라운드**: 각 라운드마다 특정 직업의 적들이 등장
- **스테이지 테마**: 스테이지 시작 시 랜덤으로 선택되며, 적의 소속(진영)을 결정

### 1.2 적 분류
적 유닛은 아군(영웅) 유닛과 동일하게 다음과 같이 분류됩니다:
- **소속 (Faction)**: 진영 시너지 (예: 노르드)
- **직업 (Class)**: 직업 시너지 (예: 처형자, 투사, 파수꾼 등)

---

## 2. 구현된 시스템

### 2.1 스테이지 테마 시스템
- 파일: `75_stage_themes.yaml`
- 스테이지 시작 시 랜덤으로 선택
- 해당 스테이지 동안 등장할 적의 소속(진영) 결정

**샘플 데이터: 노르드 진영**
```yaml
stageThemes:
  - id: 1
    name: 노르드
    faction: 3  # 노르드 진영 시너지 ID
    description: 노르드 진영의 적들이 등장합니다.
```

### 2.2 라운드 타입 시스템
- 파일: `80_round_types.yaml`
- 각 라운드마다 소환될 직업과 적 수를 정의
- 일반 라운드, 엘리트 라운드, 보스 라운드로 구분

**라운드 구성:**
- 1~3, 5~6 라운드: 일반 라운드
- 4, 8 라운드: 엘리트 라운드 (8 스테이지까지)
- 8 스테이지 이후: 7라운드 엘리트, 8라운드 보스

### 2.3 적 데이터 구조
- 파일: `15_enemies.yaml`
- 적 유닛의 모든 정보 포함 (스탯, 코드, 소속, 직업 등)

**등급 (Tier):**
- `normal`: 일반 적
- `elite`: 엘리트 적 (강화된 능력치)
- `boss`: 보스 적 (최고 능력치)

## 4. 적 배치 규칙

### 4.1 행(Row) 배치 규칙
직업에 따라 자동으로 행이 결정됩니다:

- **1행 (yPos = 1)**: 처형자, 투사, 파수꾼
- **2행 (yPos = 2)**: 책략가, 지원가, 메카닉
- **3행 (yPos = 3)**: 사수, 마법사

### 4.2 열(Column) 배치 규칙
- 적은 1~3열(cellIndex)에 배치
- 아래부터 채움 (1행부터 3행 순서로)
- 같은 행에 여러 적이 있으면 왼쪽부터 배치

## 5. 소환 프로세스

### 5.1 스테이지 시작
```csharp
_roundManager.InitializeStage(stageNumber);
```
1. 스테이지 테마 랜덤 선택
2. 적 데이터 및 라운드 타입 데이터 로드

### 5.2 라운드 시작
```csharp
_roundManager.LoadRound(roundNumber);
```
1. 현재 스테이지의 라운드 정보 로드
2. 라운드 타입에 따라 적 소환
3. 직업에 따라 적 배치
4. 시너지 계산 및 적용

### 5.3 라운드 진행
- 대기열 없음 - 모든 적은 라운드 시작 시 즉시 소환
- 적 전멸 시 라운드 종료
- 8라운드 완료 시 다음 스테이지로 진행

## 7. 테스트 가이드

### 7.1 확인사항
1. ✅ 스테이지 시작 시 테마 로그 출력
2. ✅ 라운드마다 적절한 직업의 적 소환
3. ✅ 적이 올바른 행에 배치
4. ✅ 4, 8 라운드에 엘리트 적 등장
5. ✅ 8 스테이지 이후 보스 라운드 작동
6. ✅ 8라운드 완료 시 다음 스테이지로 진행

### 7.2 디버그 로그
- 스테이지 테마 선택: `Stage X theme: 노르드`
- 적 소환: `Enemy {id} spawned at Cell ({x}, {y})`
- 라운드 시작: `Stage X, Round Y started with {roundType}`

## 9. 주의사항

⚠️ **기존 데이터 호환성**
- 기존 `70_rounds.yaml` 데이터는 더 이상 사용되지 않음
- 적 ID 범위:
  - 일반 적: 1000번대
  - 엘리트 적: 2000번대
  - 보스: 3000번대
  - 영웅: 1~999번

⚠️ **필수 데이터 파일**
- `15_enemies.yaml` - 적 데이터
- `75_stage_themes.yaml` - 스테이지 테마
- `80_round_types.yaml` - 라운드 타입 및 스테이지 구성

⚠️ **코드 의존성**
- 적 코드(패시브/일반/궁극기)가 구현되어 있어야 함
- 적 초상화 이미지가 Resources/Sprite/Portraits/에 있어야 함
