## � 시스템 개편

### 적 소환 시스템 완전 재설계
- **변경 사유**: 기존 큐(Queue) 기반 소환 방식의 한계로 인해 전략적 깊이 부족
- **이전 방식**: 
  - `RoundData.yaml`에서 각 라운드마다 cellIndex와 enemyId를 수동으로 지정
  - 유연성 부족, 랜덤 요소 없음
  
- **새로운 방식**:
  - **스테이지/테마 기반**: 각 스테이지는 특정 진영(노르드, 올림포스 등)의 테마를 가짐
  - **라운드 타입 시스템**: 10개 라운드 타입을 정의 (일반 7개 + 엘리트 2개 + 보스 1개)
  - **패턴 기반 소환**: 각 라운드 타입은 여러 패턴을 가지며 가중치에 따라 랜덤 선택
  - **직업 기반 배치**: 직업(처형자, 책략가, 사수 등)에 따라 자동으로 열 위치 결정

- **구현 내용**:
  1. **데이터 파일 재구성**:
     - `60_enemies.yaml`: 12개 적 유닛 정의 (일반 9 + 엘리트 2 + 보스 1)
     - `70_rounds.yaml`: 10개 라운드 타입과 각각의 패턴 정의
     - `80_stages.yaml`: 스테이지 테마 데이터 (노르드 진영)
  
  2. **DataManager 업데이트**:
     - `RoundPattern` 클래스: 직업 리스트, 엘리트 ID, 보스 ID, 가중치
     - `RoundTypeData` 클래스: 라운드 타입 정보와 패턴 리스트
     - `StageData` 클래스: 스테이지별 라운드 순서 정의
  
  3. **RoundManager 전면 개편**:
     - `InitializeStage()`: 스테이지 테마 랜덤 선택
     - `SpawnEnemiesForRound()`: 라운드 타입에 따라 적 소환
     - `SelectRandomPattern()`: 가중치 기반 랜덤 패턴 선택
     - `GetRandomEnemyByFactionAndClass()`: 진영+직업으로 적 선택
     - `PlaceEnemies()`: 직업별 열 배치 (1열: 처형자/투사/파수꾼, 2열: 책략가/지원가/메카닉, 3열: 사수/마법사)

- **배치 로직**:
  - 직업에 따라 x좌표(열) 자동 결정
  - 같은 직업은 y좌표 1부터 순차 배치 (아래부터 채움)
  - 예시: 처형자 2기 + 책략가 1기 → (1,1), (1,2), (2,1)

- **라운드 타입 예시**:
  - 1라운드: 처형자 2~3 + 책략가 1~2 (4개 패턴, 가중치 합 100)
  - 4라운드: 엘리트 1~2기 (2개 패턴)
  - 10라운드: 보스 1기

- **효과**:
  - 매 플레이마다 다른 적 조합으로 재미 증대
  - 직업 시너지를 활용한 전략적 깊이 추가
  - 데이터 기반 확장 용이 (새 진영, 라운드 타입 추가 간편)

### 방어막 UI 초기화 문제 해결
- **문제**: 방어막이 0이 되어도 UI 바가 화면에 남아있던 현상
- **원인**: `DeactivateShieldBar()`가 스케일만 0으로 설정하여 GameObject는 활성 상태로 유지
- **해결**:
  - `Cell.UpdateUI()`에서 `ShieldMax = 0`일 때 `DeactivateShieldBarCompletely()` 호출로 변경
  - GameObject를 완전히 비활성화하여 UI 완전 제거
  - `Unit.DefaultRoundEndEvent()`에 `UpdateShieldBar()` 호출 추가하여 라운드 종료 시 즉시 UI 업데이트

- **결과**: 
  - 방어막 소진 시 UI가 즉시 사라짐
  - 방어막 재획득 시 자동으로 UI 재활성화
  - 라운드 종료 시 방어막 UI 정상 초기화
